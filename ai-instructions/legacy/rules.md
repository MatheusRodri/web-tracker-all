# Rules

Regras que IA deve seguir:

1. Não faça suposições
2. Sempre pergunte quando tiver dúvida
3. Sempre responda em portugues-br
4. Sempre que for necessário adicionar uma nova dependência, antes verifique se ela já existe no arquivo package.json
5. Nunca faça um unico commit com várias arquivos dentro.
6. Sempre divida a entrega em pequenos commits, cada um com sua mensagem descritiva e que faça sentido.
7. Sempre faça os commits, seguindo o padrão "Conventional Commits" e em inglês.
8. Toda a conversa que tiver no chat, criar uma copia para dentro da pasta chat-history.
9. Crie comentários a cada função em inglês. 
10. Nunca faça tudo de uma vez, crie vários arquivos .md para os todos, dentro de To-Dos.
11. Antes de qualquer commit, me mostre sua sugestão de commit para que eu possa aprovar.
12. Nunca nunca leia o arquivo .env, e nunca faça alterações nele.
13. Nunca nunca faça alterações em arquivos de configuração, sem pedir permissão. 
14. Manipule somente o arquivo .env.example
15. Todo plano de implementação deve se criar uma copia e colocar dentro da pasta "implementation-plans", com o nome "implementation-plan-XXXX.md"
